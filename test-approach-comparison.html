Berlin GrillMaps - Information Architecture Test

Compare different approaches for displaying grilling regulations

Approach 1: Info Button → General Page

A prominent info button opens a side panel with all rules organized by category.

✅ Pros

  • Clean initial view
  • All info in one place
  • Easy to scan

❌ Cons

  • Extra click required
  • Not location-specific
  • May overwhelm users
View Demo →
Approach 2: Zone-Specific Detailed Popups

Rich popups with all relevant information appear when clicking on zones.

✅ Pros

  • Contextual info
  • Detailed & specific
  • Action buttons

❌ Cons

  • Large popups
  • Mobile challenging
  • Info overload
View Demo →
Approach 3: Layered Information Display

Toggle different information layers to control complexity and detail level.

✅ Pros

  • User control
  • Customizable view
  • Reduces clutter

❌ Cons

  • Learning curve
  • Hidden info
  • Complex UI
View Demo →
Approach 4: Progressive Disclosure

Information reveals gradually: hover → click → expand for more details.

✅ Pros

  • Natural flow
  • Not overwhelming
  • Mobile friendly

❌ Cons

  • Multiple clicks
  • Slower access
  • May miss info

Recommendation

For Berlin GrillMaps, Approach 4 (Progressive Disclosure) offers the best balance of accessibility, mobile-friendliness, and information depth. It allows users to quickly understand zones while providing detailed information on demand.
